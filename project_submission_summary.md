# Weather App CLI Project Summary

My project is a Python command-line application for checking the current weather and the next few days of forecast for one or more cities. The user types city names separated by commas, and the app turns that input into a complete lookup flow: first it retrieves the city's geographic coordinates using the Open-Meteo Geocoding API, and then it queries the Forecast API to get current conditions and daily forecast data. As a result, the program displays a comparison table across the requested cities and then shows the details for each one, including temperature, humidity, wind, precipitation, weather description, and a five-day forecast.

The main features of the app are support for multiple cities in a single run, side-by-side comparison in a table format, and error handling that does not interrupt the rest of the process. If one city is invalid or the API fails for a specific lookup, the remaining cities can still be processed normally. The project also includes local caching to avoid repeated calls for up to one hour and provide faster responses for recently queried cities. For the demonstration, I would show one screenshot of the terminal input with two cities, a second screenshot of the comparison table, and a third screenshot of the detailed current weather and forecast output.

I used AI as support throughout development, mainly to review the code structure, validate good practices, and speed up corrections. It was useful for discussing how to split the project into modules such as geocoding, weather lookup, display, cache, and logging, and it also helped with creating and reviewing automated tests. I also used AI to identify privacy and licensing risks, which led me to improve the project with configurable cache and logging behavior, clearer documentation, and a third-party notices file. Even with that support, I reviewed every suggestion before applying it and made sure I understood how each part worked.

What I learned most from this project was how to integrate external APIs in an organized way, validate responses before using the data, and structure a small program so it stays modular and testable. One of the biggest challenges was keeping the app simple for the user while making it robust internally, with failure handling, data validation, and readable output in the terminal. It was also challenging to think beyond the basic functionality and consider local privacy, lookup history, and correct attribution for dependencies and data sources.

One thing I am proud of is that the application delivers a clear and complete experience even as a terminal project. It does not just fetch the weather; it organizes the information in a way that is easy to compare and understand. If I had more time, I would improve the experience with command-line arguments, CSV export, and possibly a GUI or web version to make the presentation even stronger.
