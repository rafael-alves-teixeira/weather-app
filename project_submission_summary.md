# Resumo do Projeto Weather App CLI

O meu projeto e um aplicativo de linha de comando em Python para consultar o clima atual e a previsao dos proximos dias de uma ou mais cidades. O usuario digita os nomes das cidades separados por virgula e o app transforma essa entrada em um fluxo completo de consulta: primeiro busca as coordenadas geograficas da cidade usando a API de geocodificacao da Open-Meteo e, em seguida, consulta a API de previsao para obter dados atuais e previsao diaria. Como resultado, o programa mostra uma tabela comparando as cidades consultadas e, depois, exibe os detalhes de cada uma, incluindo temperatura, umidade, vento, precipitacao, descricao do tempo e previsao de cinco dias.

As principais funcionalidades do app sao a consulta de varias cidades em uma unica execucao, a comparacao lado a lado em formato de tabela e o tratamento de erros sem interromper o restante do processo. Se uma cidade estiver incorreta ou a API falhar em uma consulta, as outras ainda podem ser processadas normalmente. O projeto tambem inclui cache local para evitar chamadas repetidas por ate uma hora e resposta mais rapida para cidades pesquisadas recentemente. Para a demonstracao, eu mostraria uma captura da entrada com duas cidades, uma segunda captura da tabela comparativa no terminal e uma terceira captura do bloco detalhado com clima atual e previsao dos proximos dias.

Usei IA como apoio ao longo do desenvolvimento, principalmente para revisar a estrutura do codigo, validar boas praticas e acelerar correcoes. Ela foi util para discutir a separacao do projeto em modulos, como geocodificacao, consulta do clima, exibicao, cache e log, alem de ajudar na criacao e revisao de testes automatizados. Tambem usei IA para identificar riscos de privacidade e licenciamento, o que me levou a melhorar o projeto com configuracoes para controle de cache e log, documentacao mais clara e um arquivo de atribuicoes de terceiros. Mesmo com esse apoio, eu revisei cada sugestao antes de aplicar e mantive foco em entender o funcionamento de cada parte.

O que mais aprendi neste projeto foi como integrar APIs externas de forma organizada, validar respostas antes de usar os dados e estruturar um programa pequeno de forma modular e testavel. Um dos maiores desafios foi garantir que o app continuasse simples para o usuario, mas robusto por dentro, com tratamento de falhas, validacao de dados e saida legivel no terminal. Tambem foi desafiador pensar em aspectos que vao alem do funcionamento puro, como privacidade local, historico de consultas e atribuicao correta das dependencias e da fonte dos dados.

Uma coisa da qual eu me orgulho e que o aplicativo entrega uma experiencia clara e completa mesmo sendo um projeto de terminal. Ele nao apenas busca o clima, mas organiza as informacoes de um jeito facil de comparar e entender. Se eu tivesse mais tempo, eu melhoraria a experiencia com argumentos de linha de comando, exportacao para CSV e talvez uma versao com interface grafica ou web para deixar a apresentacao ainda mais forte.
